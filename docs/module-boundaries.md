# 模块边界检查清单

本文档定义智能客服 MVP 的后端模块和前端应用边界。新增业务代码时必须落到对应模块，不允许新增“通用业务模块”“万能服务”或跨模块直接读写对方内部数据。

## 后端模块

| 模块               | 主职责                       | 允许承担                                                       | 不允许承担                                               |
| ------------------ | ---------------------------- | -------------------------------------------------------------- | -------------------------------------------------------- |
| `conversation`     | 会话生命周期与状态流转。     | 创建会话、恢复未关闭会话、关闭会话、维护会话状态。             | 消息内容检索、AI 回答生成、坐席权限判断。                |
| `message`          | 消息归档、查询和投递适配。   | 保存访客、机器人、坐席、系统消息；按会话分页查询消息。         | 决定是否转人工、生成 AI 答案、维护知识库。               |
| `knowledge`        | 知识库运营数据管理。         | 分类、标签、FAQ、文档型知识、启停、删除、导入导出。            | 直接生成机器人回复、处理会话状态、坐席接待。             |
| `retrieval`        | 知识召回能力。               | PostgreSQL Full Text Search、pgvector、混合召回和排序。        | 知识条目运营编辑、模型调用、人工坐席流程。               |
| `ai-orchestration` | AI 问答编排。                | 调用检索、组织提示词、调用模型适配层、判断置信度和转人工建议。 | 直接维护知识库、直接写坐席接待记录、绕过适配层调用模型。 |
| `handoff`          | 转人工请求和排队状态。       | 用户主动转人工、待确认转人工、进入待接入队列、转人工系统消息。 | 坐席账号管理、AI 生成回答、报表聚合。                    |
| `agent`            | 坐席接待能力。               | 坐席状态、接入会话、人工回复、结束自己接入的会话。             | 维护知识库、生成机器人回复、管理后台账号权限。           |
| `auth`             | 登录、JWT、RBAC 和权限校验。 | 后台账号认证、角色授权、权限守卫。                             | 会话业务流转、知识检索、报表统计。                       |
| `metrics`          | 指标聚合和查询。             | 咨询量、转人工率、自动解决率、满意度、响应时长。               | 修改会话状态、维护知识正文、发送消息。                   |
| `audit`            | 审计记录。                   | 知识库变更、坐席结束会话、账号敏感操作审计。                   | 代替业务模块执行状态变更、生成业务结果。                 |

## 前端应用

| 应用            | 主职责                   | 允许承担                                                         | 不允许承担                                        |
| --------------- | ------------------------ | ---------------------------------------------------------------- | ------------------------------------------------- |
| `web-widget`    | 网站 / H5 访客客服组件。 | 聊天入口、会话初始化、访客消息、轮询消息、转人工入口、评价入口。 | 管理后台功能、坐席接待页面、直接访问数据库。      |
| `agent-console` | 坐席工作台。             | 坐席登录、待接入列表、接入会话、人工回复、结束会话。             | 知识库运营后台、访客嵌入组件、账号管理。          |
| `admin-console` | 管理后台。               | 知识库、账号、会话记录、留言、报表和系统设置。                   | 访客聊天组件、坐席实时接待 UI、直接执行业务算法。 |
| `shared-ui`     | 前端共享 UI 基础能力。   | 无业务语义的按钮、表单、布局、主题和通用反馈组件。               | 会话、知识库、坐席、权限等业务流程。              |
| `contracts`     | 前后端共享契约。         | API 类型、枚举、状态、事件和错误码。                             | 运行时业务逻辑、数据访问、UI 展示。               |

## 跨模块协作规则

- 跨模块调用必须通过公开应用服务、接口契约或事件，不直接读写其他模块内部实现。
- 数据库可以物理共库，但表结构、迁移和访问逻辑必须按模块归属组织。
- AI 模型、Embedding、对象存储等外部能力必须通过适配层调用。
- 前端共享包只放无业务语义的共享能力；业务流程留在具体应用中。
- 新增能力如果无法归属到唯一主职责模块，必须先更新本文档再实现。

## 审查清单

- 每个后端模块都有唯一主职责。
- 每个前端应用都有明确运行场景。
- 不存在集中承载所有业务逻辑的“通用业务模块”。
- 不存在绕过适配层直接调用外部 AI 服务的业务模块。
- 不存在由共享 UI 或契约包承载业务流程的情况。
