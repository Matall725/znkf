# MiMo 百万亿计划项目说明

## 项目名称

智能客服系统

## 一句话简介

面向网站和 H5 场景的 AI 智能客服系统，使用可维护知识库回答常见问题，在低置信度、敏感业务数据或用户主动要求时转接人工客服。

## 项目阶段

当前处于后端 MVP 与演示闭环阶段。项目已经完成工作区结构、TypeScript 工程规范、自动化测试框架、环境配置、数据库迁移、本地基础服务配置、后端健康检查、统一错误响应、请求日志、后台登录、RBAC 权限校验、知识分类管理、知识条目创建/编辑，以及最小知识问答接口。

## 当前可演示能力

- 后台知识运营账号可通过登录接口获取 Bearer JWT。
- 知识运营角色可创建和编辑知识分类。
- 知识运营角色可创建和编辑 FAQ/文档型知识条目。
- 访客问题可调用 `POST /api/knowledge/answer` 获取基于启用知识条目的答案。
- 未命中知识库时返回兜底提示，并标记需要转人工。
- 订单、物流、账号等具体业务数据问题不会被编造答案，会直接建议人工客服核实。
- 当前后端接口有自动化测试覆盖，包含认证、权限、知识库管理和最小问答闭环。

## MiMo Credits 使用计划

| 阶段 | 使用方向       | 预期产出                                                     |
| ---- | -------------- | ------------------------------------------------------------ |
| 1    | 知识库问答生成 | 将当前关键词检索后的原文回答升级为基于知识片段的自然语言回答 |
| 2    | 低置信度判断   | 对未命中、弱命中和敏感问题进行更稳定的转人工判断             |
| 3    | 坐席辅助       | 为人工客服生成建议回复、摘要和上下文提炼                     |
| 4    | 知识优化       | 从未解决问题中归纳知识缺口，辅助运营补全知识库               |
| 5    | 自动化验证     | 生成更多客服问答测试集，验证答案不越界、不编造业务状态       |

## 项目亮点

- 明确的人机协同边界：机器人处理常见问题，具体业务状态和低置信度问题交给人工。
- 可运营知识库：支持分类、标签、关键词、启用状态和审计记录。
- 权限边界清晰：管理员、知识库运营、坐席三类后台角色分离。
- 工程基础完整：模块边界、数据库迁移、统一错误响应、请求日志和自动化测试已经建立。
- 风险控制优先：当前问答接口只基于启用知识条目回答，敏感业务数据不生成虚假结论。

## 演示接口

### 创建知识条目

`POST /api/admin/knowledge/articles`

需要后台 Bearer JWT 和 `knowledge:write` 权限。请求体示例：

```json
{
  "articleType": "faq",
  "title": "退货政策",
  "content": "支持 7 天无理由退货，商品需保持完好并保留购买凭证。",
  "keywords": ["退货", "售后"],
  "tagNames": ["政策"],
  "status": "enabled"
}
```

### 访客提问

`POST /api/knowledge/answer`

请求体示例：

```json
{
  "question": "退货政策是什么？"
}
```

成功命中时返回示例：

```json
{
  "answer": "支持 7 天无理由退货，商品需保持完好并保留购买凭证。",
  "matched": true,
  "needsHandoff": false,
  "sourceArticle": {
    "id": "50000000-5000-4000-8000-000000000001",
    "title": "退货政策",
    "matchedTerms": ["退货"]
  }
}
```

未命中或需要人工核实时，`needsHandoff` 会返回 `true`。
